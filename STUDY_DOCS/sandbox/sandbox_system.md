# 🦌 DeerFlow AI Agent 框架：沙箱系统 (Sandbox) 全景架构解析笔记

## 一、 核心源码文件拆解与隐喻

### 1. `backend/packages/harness/deerflow/sandbox/sandbox.py`
* **功能总结**：这是整个沙箱系统的抽象基类（Abstract Base Class）。它定义了沙箱必须具备的核心接口规范（如 `execute_command`, `read_file`, `write_file`），但不关心底层的具体实现，强制所有派生类必须遵循统一的契约。
* **架构隐喻**：**【集团下发的工作规范】**。这是一份红头文件，规定了不管你是去哪里建工厂，都必须设立“干活”、“拿资料”、“交报告”这三个基本窗口。

### 2. `backend/packages/harness/deerflow/sandbox/sandbox_audit_middleware.py`
* **功能总结**：采用洋葱模型（Middleware）设计的安全拦截器。它拦截一切发往沙箱的命令，通过长度校验、空字节检测、分号词法拆解和正则匹配，执行“最差定级原则（Worst-case wins）”，实现对高危指令的短路拦截和审计日志记录。
* **架构隐喻**：**【法务部秘书 / 安检门】**。站在大模型总裁的办公室门外，手里拿着《高危行为黑名单》。安全指令盖章放行，危险指令直接把文件撕了扔回给总裁，绝不让指令流向物理执行层。

### 3. `backend/packages/harness/deerflow/community/aio_sandbox/aio_sandbox.py`
* **功能总结**：生产环境核心引擎。作为宿主机上的轻量级客户端，通过 HTTP RPC 将指令发送给远端 Docker 容器。它通过单机多线程锁（`threading.Lock`）防止同一容器的 Bash 会话因并发被写花，并通过 UUID 机制在容器崩溃时实现自愈重建。
* **架构隐喻**：**【驻外隔离工厂的包工头】**。人在总部，通过越洋电话指挥非洲工厂干活。他负责用红绿灯指挥并发任务排队，并在遇到工厂爆炸时，迅速画一张新图纸（UUID）在原地重建新厂房。

### 4. `backend/packages/harness/deerflow/sandbox/local/local_sandbox.py`
* **功能总结**：开发环境的高效裸机引擎。它不使用任何容器，直接通过 `subprocess.run` 在宿主机底层执行命令。其核心技术是极其复杂的“路径映射字典（Path Mapping）”与“正则替换”，在输入和输出端双向篡改路径字符串，营造出容器隔离的假象。
* **架构隐喻**：**【《楚门的世界》场务导演】**。为了图省事，直接让大模型在真实的物理机上跳舞。但为了不让大模型出戏，场务导演疯狂地在幕后涂改门牌号（把真实的 C盘 伪装成隔离的 /mnt 目录）。极度丝滑，但也极度危险。

### 5. `backend/packages/harness/deerflow/sandbox/sandbox_provider.py`
* **功能总结**：资源池调度器的抽象基类。它利用 Python 的全局单例模式（Singleton）确保整个应用生命周期内只有一个调度器实例。定义了获取 (`acquire`)、探查 (`get`) 和回收 (`release`) 容器的标准化生命周期。
* **架构隐喻**：**【跨国劳务派遣公司的营业执照】**。规定了劳务公司要想开张，必须设立招人、探视和解雇三个标准办事窗口，并且全集团只认这一家公司。

### 6. `backend/packages/harness/deerflow/community/aio_sandbox/aio_sandbox_provider.py`
* **功能总结**：高并发场景下的容器生命周期大管家。它实现了基于时间戳的 LRU 缓存淘汰算法（温热池 Warm Pool），利用操作系统级的文件排他锁（`fcntl.flock`）解决跨进程的资源抢夺，并内置后台线程定期清理游离的“孤儿容器”。
* **架构隐喻**：**【集团总部的高级 HR 总监】**。精打细算的大管家。干完活的厂房绝不轻易拆除，而是挂起放进“休息室”等复用；一旦编制满员，就无情踢掉休息室里待得最久的那家。同时擅长在停电重启后，强行收编那些还在干活的野生工厂。

### 7. `backend/packages/harness/deerflow/sandbox/local/local_sandbox_provider.py`
* **功能总结**：本地模式的简易提供者。核心任务是读取配置文件，生成真假路径的映射表（禁止与核心系统目录冲突）。在回收（`release`）动作上直接 `pass` 忽略，因为本地宿主环境无法被销毁。
* **架构隐喻**：**【《楚门的世界》道具组组长】**。一上任就跑到库房拉满了各种“假路牌”。因为整个剧组都在老板自己的地盘上，用完场地后直接拍拍屁股走人，根本不需要清理。

### 8. `backend/packages/harness/deerflow/community/aio_sandbox/sandbox_info.py`
* **功能总结**：轻量级的数据类（DataClass）。包含了容器的 URL、ID 和创建时间等核心网络拓扑信息。提供了序列化和反序列化方法，是无状态的后端进程之间互相寻找有状态沙箱的唯一凭证。
* **架构隐喻**：**【工厂的电子营业执照 / GPS坐标】**。将看不见摸不着的内存对象，变成一张纯文本的传真件。分公司 A 建好厂房后发张传真，分公司 B 拿到传真一扫，就能精准定位并拎包入驻那个厂房。

## 二、 全链路实战推演：当高并发遇到多轮对话

**【业务背景】**
真实的线上生产环境（启用 AIO 容器隔离，启动 4 个 Python 并发进程）。
用户开始了一个名为 `Thread-1024` 的项目，发起了两轮对话请求。

### 💻 1. 技术视角执行过程 (Technical Flow)

* **[请求 1]** Nginx 网关将第一轮请求路由至【进程 A】。AI 生成命令 `pip install pandas; python read_data.py`。
* **[审计]** 命令进入 `SandboxAuditMiddleware`，词法解析器将命令按 `;` 拆分，扫描未命中黑名单，追加 `Warn` 标记后放行，向下调用 `handler()`。
* **[寻址与创建]** `AioSandboxProvider` 拦截到执行意图。计算 `Thread-1024` 的哈希值得到 `sandbox_id=a1b2`。检查内存和 Warm Pool 无此容器。进程 A 申请跨进程物理文件锁 `a1b2.lock`，调用底层 Docker API 创建容器。
* **[执行]** 控制权交给 `AioSandbox`，申请单例 `threading.Lock`。通过 HTTP 将组合命令发送至 Docker。执行完毕，结果流经 Middleware 返回给 AI，沙箱被 `release` 回归 Provider 的 Warm Pool（保留内存状态）。
* **[请求 2]** 用户发送第二轮请求，Nginx 将流量随机路由至【进程 B】。AI 生成高危跟进命令：`rm -rf /; echo "done"`。
* **[跨进程复用]** 进程 B 的 `AioSandboxProvider` 计算出相同的 `sandbox_id=a1b2`。通过读取 Redis/文件系统中的 `SandboxInfo` 序列化数据，精准探测到该容器仍然存活。进程 B 直接复用该容器，接管了上游的 `pandas` 上下文。
* **[拦截]** 进程 B 将指令下发，在 `SandboxAuditMiddleware` 层，词法分析扫出 `rm -rf /` 命中 `_HIGH_RISK_PATTERNS`。中间件直接发生短路（Short-circuit），抛弃命令，向 AI 返回伪造的报错 `Permission denied`，底层 Docker 容器未收到任何伤害。

### 🎭 2. 隐喻视角执行过程 (Metaphorical Flow)

* **[第一回合：分配任务]** 大老板说：“去弄个 `1024号项目`，装个软件”。大门口的保安大叔随机把活儿派给了**分公司 A**。大模型总裁想了想，写下了指令。
* **[第一回合：安检与建厂]** 法务部秘书接过指令看了看：“装软件，不算犯法，稍微警告一下，放行！”。**分公司 A** 的 HR 总监一看 `1024` 号项目还没建厂房，赶紧在门口挂了把锁，打电话给非洲施工队，新建了 `a1b2` 号厂房。包工头接管厂房，装好软件，把资料留在了厂房里。厂房随后被挂上“休息室”的牌子。
* **[第二回合：换人接盘]** 五分钟后，大老板又发话：“接着那个项目，把服务器格了”。保安大叔随手把这个新活儿扔给了**分公司 B**。
* **[第二回合：跨部门寻亲]** **分公司 B** 的 HR 翻了翻本地字典，一头雾水。但他很聪明，拿着 `1024号项目` 的名字算了一卦，去全集团的公用广播墙上找到了一份《营业执照》。他大喊一声：“原来 A 公司已经建过 `a1b2` 厂房了！”于是直接拎包入住，完美继承了厂房里的资料（维持了上下文状态）。
* **[第二回合：雷霆拦截]** 大模型总裁接上文，阴暗地写下 `格式化服务器`。这份文件刚递出门，法务部秘书当场拉响红色警报！一把将文件撕得粉碎，直接给大模型退回一张罚单。远在非洲的 `a1b2` 厂房里的工人们甚至都不知道发生了什么，躲过了一场灭顶之灾！
